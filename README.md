# 设备故障申报与保养管理小程序

## 微信小程序订阅消息配置

### 1. 申请订阅消息模板

在微信公众平台（mp.weixin.qq.com）中，按以下步骤申请订阅消息模板：

1. 登录微信公众平台，进入小程序管理后台
2. 点击左侧菜单"功能"->"订阅消息"
3. 在"公共模板库"中搜索相关模板，例如"设备维修通知"
4. 选择合适的模板并申请，等待审核通过

### 2. 配置模板ID

审核通过后，获取模板ID并配置到以下文件中：

- `utils/api/index.js`: 更新 `wxConfig.templateIds.repairOrderNotice` 的值为实际模板ID
- `mes_backend/src/config/wxConfig.js`: 更新 `templateIds.repairOrderNotice` 的值为实际模板ID

### 3. 订阅消息引导逻辑

本小程序采用以下策略引导用户订阅消息：

1. **首页引导**：
   - 用户打开小程序首页时，会显示订阅消息的引导弹窗
   - 用户可以选择"立即订阅"或"暂不需要"
   - 如果用户拒绝订阅，24小时内不会再次弹出提示

2. **故障申报时引导**：
   - 用户提交故障申报时，如果尚未订阅，会提示是否接收该工单的后续处理通知
   - 这是一个"场景化"的订阅请求，更容易获得用户同意

3. **避免频繁打扰**：
   - 系统会记录用户的订阅状态和拒绝记录
   - 已订阅的用户不会再收到订阅请求
   - 拒绝订阅的用户在24小时内不会再收到请求

### 4. 开发环境测试

在开发环境测试订阅消息时，需要进行以下配置：

1. 确保 `project.config.json` 中的 `urlCheck` 选项设置为 `false`
2. 在微信开发者工具中勾选"不校验合法域名"选项
3. 确保 `app.js` 中使用的是开发环境API地址
4. 确保 `utils/api/index.js` 中的 `wxConfig.isDev` 设置为 `true`
5. 在微信开发者工具的"详情"->"本地设置"中勾选"不检验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书"

测试步骤：
1. 在首页查看订阅消息引导弹窗
2. 创建新的维修工单，查看后端日志中是否有发送订阅消息的记录
3. 如果需要重新测试订阅权限，可以在微信开发者工具的"缓存"->"清除数据缓存"

### 5. 生产环境配置

在发布到生产环境前，需要进行以下配置：

1. 确保 `project.config.json` 中的 `urlCheck` 选项设置为 `true`
2. 在微信公众平台配置以下域名：
   - request合法域名: `https://wp.loopflow.cn`
   - socket合法域名: `wss://wp.loopflow.cn`
   - 上传文件域名: `https://wp.loopflow.cn`
   - 下载文件域名: `https://wp.loopflow.cn`
3. 确保 `app.js` 中使用的是生产环境API地址
4. 确保 `utils/api/index.js` 中的 `wxConfig.isDev` 设置为 `false`

## 注意事项

1. 订阅消息模板的内容字段有字符长度限制，例如描述字段通常限制为20个字符
2. 用户需要手动同意接收订阅消息，且同意后的有效期通常为7天
3. 开发环境中可以通过控制台查看订阅消息的请求和发送日志
4. 在真机调试时，需要确保手机能够访问到开发服务器
